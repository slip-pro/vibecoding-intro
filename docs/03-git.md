# Git и GitHub

> AI-помощник проведёт тебя через это автоматически.
> Здесь -- справочник для самостоятельного чтения.

**⏱ Время: ~20 минут**

Сейчас весь код твоего сайта хранится только на твоём
компьютере. Если случайно удалишь файл, переустановишь
систему или жёсткий диск выйдет из строя -- всё пропадёт.
Никаких резервных копий, никакой истории изменений.

Git решает эту проблему. А GitHub -- хранит твой код
в облаке, где он в безопасности.

---

## Часть 1: Что такое Git

**⏱ ~2 минуты на чтение**

**Git -- это машина времени для кода.** Он позволяет
сохранять "снимки" проекта в определённые моменты
и возвращаться к любому из них. Сломал что-то? Откатись
к предыдущему снимку. Хочешь вспомнить, как выглядел сайт
вчера? Легко.

Три ключевых понятия:

### Коммит (commit)

Коммит -- это снимок проекта в определённый момент времени.
Как фотография: ты нажимаешь кнопку -- и текущее состояние
всех файлов сохраняется навсегда. Каждый коммит получает
уникальный номер и подпись: кто сделал, когда, и короткое
описание ("Добавил шапку сайта", "Исправил цвет кнопки").

> Аналогия: представь, что ты рисуешь картину. Каждый час
> ты фотографируешь холст. Если случайно закрасишь удачный
> фрагмент -- можно посмотреть фото и восстановить. Коммит --
> это такая фотография для кода.

### Репозиторий (repository)

Репозиторий (или "репо") -- это папка проекта, за которой
следит Git. Внутри неё Git хранит все коммиты -- всю историю
изменений. Если коммит -- это фотография, то репозиторий --
это фотоальбом со всеми снимками проекта от начала до конца.

### Push (пуш)

Push -- это отправка коммитов из твоего компьютера в облако
(на GitHub). Как загрузка фотографий в Google Photos: снимки
теперь не только у тебя на телефоне, но и в интернете.
Даже если потеряешь телефон -- фотографии в безопасности.
То же самое с кодом: после push он хранится и у тебя,
и на GitHub.

## Часть 2: Установка Git

**⏱ ~5 минут**

### Проверка: может, уже установлен?

Открой терминал в Cursor (**Ctrl+`** -- это кнопка
с обратной кавычкой, левее цифры 1) и набери:

```
git --version
```

Если видишь что-то вроде `git version 2.43.0` -- Git уже
установлен, переходи сразу к **Части 3**.

Если видишь ошибку "git не является внутренней или внешней
командой" -- нужно установить.

### Установка

**Windows:**

1. Открой в браузере сайт [git-scm.com](https://git-scm.com)
2. Нажми большую кнопку **"Download for Windows"**
3. Открой скачанный файл двойным кликом
4. На большинстве экранов просто нажимай **"Next"**
5. **Важный момент:** на экране "Choosing the default editor
   used by Git" выбери **"Use Visual Studio Code as Git's
   default editor"**
6. Остальные экраны → **"Next"** → **"Install"** → **"Finish"**

**macOS:**

На macOS Git часто уже установлен. Если команда `git --version`
выше выдала ошибку, система сама предложит установить
Command Line Tools -- просто нажми **"Установить"** в диалоговом
окне и подожди. Если диалог не появился:

1. Открой терминал и выполни:
   ```
   xcode-select --install
   ```
2. Нажми **"Установить"** в появившемся окне
3. Дождись окончания (может занять 5-10 минут)

### После установки

**Обязательно перезапусти Cursor** (закрой и открой заново).
Это нужно, чтобы терминал "увидел" новую программу.

Снова открой терминал (**Ctrl+`**) и проверь:

```
git --version
```

Теперь должна появиться версия. Если появилась -- отлично,
Git установлен.

### Настройка имени и email

Git подписывает каждый коммит -- как автограф на фотографии.
Нужно один раз указать своё имя и email. Набери в терминале
(подставь свои данные):

```
git config --global user.name "Твоё Имя"
git config --global user.email "твой@email.com"
```

Например:

```
git config --global user.name "Anna Ivanova"
git config --global user.email "anna@example.com"
```

Это делается один раз -- Git запомнит и будет использовать
для всех будущих проектов.

> **⚠️ Если Git не устанавливается**
>
> - **Windows: ошибка при установке** -- нажми правой кнопкой
>   мыши на установщик, выбери "Запуск от имени администратора"
> - **macOS: xcode-select зависает** -- проверь интернет,
>   при необходимости перезапусти команду
> - **После установки команда не найдена** -- убедись, что
>   перезапустил Cursor. Если не помогло -- перезагрузи
>   компьютер

## Часть 3: Первый коммит

**⏱ ~5 минут**

Теперь сделаем первый "снимок" твоего проекта. Убедись,
что терминал Cursor открыт в папке твоего сайта (my-site
или как ты его назвал). В терминале должен быть путь вроде
`D:\Projects\my-site` (Windows) или `~/Projects/my-site`
(macOS).

> Если терминал открыт в другой папке -- набери:
> ```
> cd путь/к/my-site
> ```
> (подставь свой путь)

### Шаг 1: Инициализация

```
git init
```

Эта команда говорит Git: "Следи за этой папкой". Git создаст
скрытую папку `.git` внутри проекта -- это его "память",
где он будет хранить все снимки. Ты не увидишь эту папку
в обычном проводнике, но она есть.

### Шаг 2: Добавление файлов

```
git add .
```

Точка означает "все файлы в текущей папке". Эта команда
говорит Git: "Подготовь все файлы для снимка". Это как
расставить всех на групповом фото -- пока не щёлкнул
затвором, можно ещё кого-то добавить или убрать.

### Шаг 3: Создание коммита

```
git commit -m "Мой первый сайт"
```

Вот теперь снимок сделан! Флаг `-m` означает "message" --
сообщение, описание коммита. В кавычках пишешь что сделал.
В будущем это поможет понять, что было в каждом снимке:
"Добавил шапку", "Поменял цвета", "Исправил мобильную
версию".

<!-- screenshot: терминал с git commit -->

### Результат

Если всё прошло успешно, ты увидишь что-то вроде:

```
[main (root-commit) abc1234] Мой первый сайт
 15 files changed, 500 insertions(+)
```

Числа будут другими -- это нормально. Главное, что нет
красных ошибок. Поздравляю -- первый снимок сохранён!

> **⚠️ Если git commit выдаёт ошибку**
>
> - **"Please tell me who you are"** -- ты не настроил имя
>   и email. Вернись к разделу "Настройка имени и email"
>   в Части 2 и выполни команды `git config`
> - **"nothing to commit"** -- возможно, ты уже делал коммит
>   раньше или файлов нет в папке. Набери `git status` чтобы
>   посмотреть состояние

## Часть 4: GitHub

**⏱ ~10 минут**

Git сохраняет снимки на твоём компьютере. Но если с компьютером
что-то случится -- они пропадут. GitHub -- это облачное
хранилище для кода. Как Google Drive, только специально
для проектов.

Когда код на GitHub:
- Он в безопасности (хранится на серверах Microsoft)
- Ты можешь получить к нему доступ с любого компьютера
- Другие люди смогут посмотреть твой проект (если захочешь)
- Это пригодится для деплоя (публикации сайта)

### Регистрация на GitHub

Если у тебя уже есть аккаунт -- пропусти этот раздел.

1. Открой в браузере [github.com](https://github.com)
2. Нажми **"Sign up"** (Зарегистрироваться)
3. Введи email, придумай пароль, выбери имя пользователя
   (username)
4. Пройди проверку (решение картинки-головоломки)
5. Подтверди email -- зайди в почту, найди письмо
   от GitHub, нажми кнопку подтверждения

> Имя пользователя (username) станет частью адреса
> твоих проектов: `github.com/ТВОЙ-ЛОГИН/my-site`.
> Выбирай что-то читаемое и приличное.

<!-- screenshot: регистрация на GitHub -->

### Создание репозитория

Теперь создадим место на GitHub, куда отправим код.

1. Войди в свой аккаунт на [github.com](https://github.com)
2. В правом верхнем углу нажми **"+"** (плюс)
3. Выбери **"New repository"** (Новый репозиторий)
4. Заполни:
   - **Repository name:** `my-site` (или как назвал проект)
   - **Description:** `Мой первый сайт` (необязательно,
     но приятно)
   - **Выбери Private** (приватный -- только ты видишь)
5. **НЕ ставь галочки** на "Add a README file", "Add
   .gitignore", "Choose a license" -- у тебя уже всё есть
   в проекте
6. Нажми **"Create repository"** (Создать репозиторий)

<!-- screenshot: создание репозитория на GitHub -->

> **Почему Private?** Приватный репозиторий видишь только ты.
> Потом, если захочешь, всегда можно сделать его публичным
> в настройках. Но пока -- пусть будет только для тебя.

### Связывание и отправка

После создания репозитория GitHub покажет страницу
с инструкциями. Тебе нужно выполнить три команды
в терминале Cursor (в папке проекта):

```
git remote add origin https://github.com/ТВОЙ-ЛОГИН/my-site.git
git branch -M main
git push -u origin main
```

**Замени** `ТВОЙ-ЛОГИН` на своё имя пользователя на GitHub,
а `my-site` -- на название своего репозитория.

Что делает каждая команда:

- `git remote add origin ...` -- говорит Git: "Вот адрес
  облачного хранилища, называй его origin". Это как
  добавить адрес Google Photos, чтобы Git знал куда
  загружать снимки.
- `git branch -M main` -- называет текущую ветку "main".
  Не вдавайся в детали -- это стандартная настройка.
- `git push -u origin main` -- отправляет все коммиты
  на GitHub. "Push" = толкнуть = отправить в облако.
  Флаг `-u` запоминает связь, чтобы в будущем можно было
  писать просто `git push`.

### Авторизация

При первом push GitHub спросит, кто ты такой. Есть два
варианта, как это может выглядеть:

**Вариант А: всплывающее окно браузера.** Откроется страница
GitHub с просьбой подтвердить доступ. Просто нажми
**"Authorize"** -- это самый простой способ.

**Вариант Б: запрос логина и пароля в терминале.** Если
терминал просит пароль -- обычный пароль от GitHub
**не подойдёт**. Нужен Personal Access Token (персональный
токен доступа):

1. Открой в браузере:
   [github.com/settings/tokens](https://github.com/settings/tokens)
2. Нажми **"Generate new token"** → **"Generate new token
   (classic)"**
3. В поле **"Note"** напиши что-то вроде: `Cursor`
4. В **"Expiration"** выбери срок (например, 90 дней)
5. Поставь галочку на **"repo"** (доступ к репозиториям)
6. Нажми **"Generate token"** внизу страницы
7. **Скопируй токен** (длинная строка из букв и цифр).
   **Важно:** он покажется только один раз!
8. Вставь этот токен вместо пароля в терминале

> Чтобы не вводить токен каждый раз, набери в терминале:
> ```
> git config --global credential.helper store
> ```
> Git запомнит токен после первого ввода.

### Проверка

Открой в браузере свой репозиторий:

```
https://github.com/ТВОЙ-ЛОГИН/my-site
```

Видишь свои файлы? Код проекта отображается на странице?
Значит всё получилось -- твой сайт теперь в безопасности
в облаке.

<!-- screenshot: код проекта на GitHub -->

> **⚠️ Если push не работает**
>
> - **"Authentication failed"** -- проблема с авторизацией.
>   Попробуй создать новый Personal Access Token (инструкция
>   выше)
> - **"remote origin already exists"** -- ты уже добавлял
>   remote. Набери `git remote set-url origin НОВЫЙ-URL`
>   чтобы обновить адрес
> - **"failed to push"** -- убедись, что при создании
>   репозитория ты НЕ ставил галочки на README, .gitignore,
>   license. Если поставил -- проще удалить репозиторий
>   на GitHub и создать заново
> - **Ничего не происходит, терминал завис** -- возможно,
>   он ждёт пароль. Попробуй ввести токен и нажать Enter
>   (при вводе пароля символы не отображаются -- это нормально)

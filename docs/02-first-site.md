# Создание первого сайта

> AI-помощник проведёт тебя через всё это автоматически.
> Эта страница -- справочник на случай если хочешь понять
> что происходит или пройти путь самостоятельно.

**⏱ Время: ~1.5 часа** (установка + создание + разработка)

---

## Часть 1: Установка Node.js

**⏱ ~10 минут**

### Что это такое

Node.js -- это движок для сайтов. Как мотор для машины --
без него сайт не заведётся. Все современные сайты используют
Node.js для сборки и запуска. Тебе не нужно понимать как он
работает внутри -- просто установи его, и всё заработает.

npm -- это менеджер пакетов, который идёт вместе с Node.js.
Думай о нём как об App Store для программных библиотек.
Когда нужна какая-то готовая функция -- npm скачает её
за тебя одной командой.

### Проверка: может, уже установлен?

Прежде чем устанавливать -- проверь, вдруг Node.js уже есть:

1. Открой терминал в Cursor: нажми **Ctrl+`** (клавиша
   с обратной кавычкой, слева от цифры 1)
2. Набери команду и нажми Enter:
   ```
   node --version
   ```
3. Если увидишь что-то вроде `v20.11.0` -- Node.js уже
   установлен, переходи к Части 2!
4. Если увидишь ошибку -- нужно установить

<!-- screenshot: терминал с node --version -->

### Установка

1. Открой в браузере [nodejs.org](https://nodejs.org)
2. На главной странице будут две большие кнопки. Нажми ту,
   что зелёного цвета с надписью **"LTS"** (это стабильная
   версия, рекомендованная для большинства пользователей)
3. Дождись окончания скачивания
4. Открой скачанный файл двойным кликом
5. Появится окно установки. Нажимай **"Далее"** ("Next")
   на каждом шаге -- настройки по умолчанию подходят
6. На последнем экране нажми **"Установить"** ("Install")
7. Дождись окончания
8. Нажми **"Готово"** ("Finish")
9. **Важно:** полностью закрой Cursor и открой его заново
   (это нужно чтобы терминал "увидел" новую программу)

<!-- screenshot: сайт nodejs.org -->

### Проверка после установки

Открой терминал в Cursor (**Ctrl+`**) и выполни две команды:

```
node --version
```

```
npm --version
```

Обе должны вернуть номер версии (например, `v20.11.0`
и `10.2.4`). Конкретные цифры не важны -- главное,
что ошибки нет.

<!-- screenshot: терминал с node --version и npm --version -->

> **⚠️ Если после установки команды не работают**
>
> - Полностью закрой Cursor (не просто окно, а именно
>   выйди из программы) и открой заново
> - Если всё ещё не работает -- **перезагрузи компьютер**
>   целиком. Иногда системе нужна перезагрузка чтобы
>   "увидеть" новые программы
> - После перезагрузки открой Cursor и попробуй снова

## Часть 2: Создание проекта Next.js

**⏱ ~15 минут**

### Что такое Next.js

Next.js -- это конструктор для сайтов. Как LEGO -- из готовых
блоков собираешь что хочешь. Тебе не нужно создавать всё
с нуля: Next.js даёт базовую структуру, а ты (вместе
с AI-помощником) наполняешь её содержимым.

Тысячи настоящих компаний используют Next.js для своих
сайтов: от стартапов до крупных сервисов. Это
профессиональный инструмент, но при этом достаточно простой,
чтобы начать с ним работать прямо сейчас.

### Создание проекта

1. Открой терминал в Cursor (**Ctrl+`**)
2. Выполни команду:
   ```
   npx create-next-app@latest my-site
   ```
   `npx` -- это утилита, которая скачает и запустит
   создание проекта. `my-site` -- название папки
   с твоим будущим сайтом. Можешь заменить на своё,
   например `my-coffee-shop` или `portfolio`.

3. Программа задаст несколько вопросов. Вот что отвечать
   на каждый:

| Вопрос | Ответ | Зачем |
|--------|-------|-------|
| Would you like to use TypeScript? | **Yes** | Помогает находить ошибки в коде |
| Would you like to use ESLint? | **Yes** | Автоматически проверяет код на ошибки |
| Would you like to use Tailwind CSS? | **Yes** | Удобный способ делать красивые стили |
| Would you like your code inside a `src/` directory? | **No** | Проще структура для начала |
| Would you like to use App Router? | **Yes** | Современный подход к навигации |
| Would you like to use Turbopack? | **Yes** | Ускоряет сборку во время разработки |
| Would you like to customize the import alias? | **No** | Стандартные настройки подходят |

   Не волнуйся, если не понимаешь что значит каждый
   пункт -- просто отвечай как в таблице. AI-помощник выбрал
   эти настройки чтобы тебе было проще.

4. Дождись пока всё установится (может занять 1-2 минуты,
   в терминале будут бежать строчки -- это нормально)

5. Когда установка закончится, перейди в папку проекта:
   ```
   cd my-site
   ```

<!-- screenshot: терминал с create-next-app -->

### Структура проекта

После создания у тебя появится папка с файлами.
Вот что внутри (объясню только важное):

```
my-site/
├── app/           ← Здесь живут страницы сайта
│   ├── page.tsx   ← Главная страница
│   ├── layout.tsx ← "Обёртка" для всех страниц
│   └── globals.css ← Стили (цвета, шрифты, отступы)
├── public/        ← Картинки, иконки, файлы
├── package.json   ← Список зависимостей (как список ингредиентов в рецепте)
└── ...            ← Остальное пока не важно
```

Главное что нужно знать:
- **app/page.tsx** -- это то, что видит посетитель сайта.
  Именно этот файл мы будем менять
- **public/** -- сюда кладутся картинки, которые ты хочешь
  показать на сайте
- Остальные файлы -- служебные, их трогать не нужно

### Запуск сайта

1. Убедись что ты в папке проекта (после команды `cd my-site`
   выше). В терминале должен быть путь к твоему проекту
   (например, `C:\...\my-site>` или `~/Projects/my-site`)
2. Выполни команду:
   ```
   npm run dev
   ```
3. В терминале появится что-то вроде:
   ```
   ▲ Next.js 15.x.x
   - Local: http://localhost:3000
   ```
4. Открой в браузере адрес: **http://localhost:3000**
5. Видишь страницу? Это твой сайт!

<!-- screenshot: стандартная страница Next.js в браузере -->

Пока это стандартный шаблон -- чёрная страница с логотипом
Next.js. Не переживай, скоро мы превратим его в красивый
сайт с твоим дизайном.

> **⚠️ Если localhost:3000 не открывается**
>
> - Проверь что команда `npm run dev` всё ещё работает
>   в терминале. Если ты случайно закрыл терминал
>   или нажал Ctrl+C -- команда остановилась.
>   Просто выполни `npm run dev` снова.
> - Попробуй другой браузер
> - Убедись что адрес написан правильно:
>   `http://localhost:3000` (без https, без www)

> **⚠️ Как остановить и запустить снова**
>
> Когда захочешь остановить сайт -- нажми **Ctrl+C**
> в терминале. Чтобы снова запустить -- просто выполни
> `npm run dev`. Можешь останавливать и запускать
> сколько угодно раз.

## Часть 3: Разработка с AI-помощником

**⏱ ~60 минут**

### Самое интересное

Теперь начинается магия -- превращаем пустой шаблон
в твой собственный сайт! Тебе не нужно писать ни одной
строчки кода вручную. Просто разговаривай с AI-помощником
на обычном русском языке.

### Как это работает

Процесс очень простой -- работаешь итерациями:

1. **Опиши что хочешь** -- тему сайта, цвета, какие
   секции нужны
2. **AI-помощник создаст первую версию** -- напишет код
   и изменит файлы проекта
3. **Посмотри в браузере** -- страница обновится
   автоматически (если `npm run dev` запущен). Иногда
   нужно просто обновить страницу клавишей F5
4. **Скажи что нравится, а что нет** -- "текст слишком
   мелкий", "хочу другой цвет фона", "добавь ещё секцию"
5. **AI-помощник исправит** -- внесёт правки по твоим пожеланиям
6. **Повторяй** -- пока не будешь доволен результатом

Каждый круг такой итерации занимает пару минут. За час
можно сделать 10-20 итераций -- этого достаточно чтобы
получить красивый, законченный сайт.

### Примеры промптов

Вот что можно сказать AI-помощнику на разных этапах:

**Создание с нуля:**
- "Создай лендинг для кофейни с тёплыми цветами,
  секции: герой, меню, о нас, контакты"
- "Сделай персональный сайт-портфолио для фотографа
  в минималистичном стиле с большими фото"
- "Создай лендинг для мобильного приложения с градиентным
  фоном и анимациями"

**Правки и улучшения:**
- "Сделай текст крупнее и добавь больше пространства
  между секциями"
- "Поменяй фон на тёмный с белым текстом"
- "Кнопка 'Связаться' должна быть ярче и заметнее"
- "Добавь красивую анимацию при прокрутке"

**Адаптивность:**
- "Сделай чтобы хорошо выглядело на телефоне"
- "На мобильном меню должно быть в виде гамбургера"

**Контент:**
- "Замени текст-заглушку на настоящий текст для кофейни
  'Утренний свет'"
- "Добавь секцию с отзывами клиентов"
- "Добавь карту проезда в секцию контактов"

### Советы для хорошего результата

**Будь конкретным.** Чем подробнее ты опишешь что хочешь --
тем точнее будет результат. Вместо "сделай красиво"
лучше сказать "хочу светлый фон, синие акценты,
крупный заголовок по центру и картинку справа".

**Не стесняйся просить много раз.** Итерации -- это
нормальный процесс. Даже профессиональные дизайнеры
делают десятки правок. Попросил -- посмотрел --
попросил исправить -- это правильный путь.

**Если результат совсем не тот -- начни описание заново.**
Иногда проще сказать "давай начнём сначала, хочу
совсем другое" и описать заново более подробно, чем
пытаться исправить то что получилось.

**Описывай ощущения.** Можно говорить не только про конкретные
вещи, но и про настроение: "слишком скучно, хочу
динамичнее", "выглядит дёшево, хочу более premium",
"слишком пёстро, хочу спокойнее". AI-помощник поймёт.

**Смотри на телефоне.** Открой тот же адрес
`http://localhost:3000` на телефоне (если он в одной
Wi-Fi сети с компьютером -- вместо localhost
используй IP-адрес компьютера). Или в браузере нажми
**F12**, потом иконку телефона/планшета вверху --
это режим мобильной версии.
